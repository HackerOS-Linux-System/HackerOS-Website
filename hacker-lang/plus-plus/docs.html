v0.1.0 OOP JIT x86_64 Arc<> memory HackerOS

Wprowadzenie

Hacker Lang ++ (HL++) to interpretowany język programowania napisany w Rust, będący bezpośrednim następcą Hacker Lang (HL). Zachowuje całą unikalną składnię HL i rozszerza ją o pełne programowanie obiektowe.

|> Klasy new in HL++

|> Animal
    ~name  = Unknown     ;; deklaracja pola z domyślną wartością
    ~sound = ...
    ~alive = true

    => new(n, s) def      ;; konstruktor
        @self.name  = @n
        @self.sound = @s
    done

    => speak def
        ~> @self.name says: @self.sound
    done
done

;; Tworzenie instancji
% dog = Animal::new(Rex, Woof!)
-- @dog.speak

|>> Dog : Animal

    ~tricks = 0

    => new(n, b) def

        @self.breed = @b
    done

    => learn_trick def
        @self.tricks = @self.tricks + 1
        ::green @self.name learned trick #@self.tricks!
    done
done

*{} Atrybuty HL++ only

HL++ posiada własny system atrybutów inspirowany Rust #[], ale zgodny ze składnią operatorową:

;; HL++ attribute syntax: *{name(args)}
*{derive(Debug, Clone)}
|> MyClass
    ~field = 0
done

*{deprecated(since=0.2.0, use=new_fn)}
: old_function def
    ~> Ten kod jest przestarzały
done

*{inline}
: hot_path(n) def
    ~> Wynik: @n
done
hlpp
Porównanie z Rust

Rust używa #[derive(Debug)] — HL++ używa *{derive(Debug)} (gwiazdka zamiast kratki, klamra zamiast nawiasu kwadratowego).

~| Extern System new in HL++

HL++ posiada wbudowany system integracji z bibliotekami Java, Perl, Go i JavaScript. Symbol ~| (tilde-pipe) — wybrany jako jedyny wolny spójny z unikalną składnią HL++.

;; Java — via jshell lub groovy
~|java import "java.util.ArrayList"
~|java "Math"          ::sqrt(16.0)
~|java "StringBuilder"  ::new()
~|java "System.out"     ::println("Hello!")

;; Perl — via perl interpreter
~|perl import "List::Util"
~|perl "POSIX"       ::floor(3.7)
~|perl "List::Util"  ::max(1, 5, 3, 9)
~|perl "List::Util"  ::sum(1, 2, 3, 4)

;; Go — via go run (tymczasowy plik)
~|go import "strings"
~|go "math"     ::Sqrt(25.0)
~|go "strings"  ::ToUpper("hello")

;; JavaScript — via Node.js (najbardziej natywny)
~|js import "lodash" as _

Dlaczego ~| ?

Symbol ~| (tilde-pipe) był jedynym wolnym dwuznakiem w składni HL++. Tilde (~) już był używany do pól klas (~field) i printa (~>), więc ~| jako "tilde-pipe = extern bridge" jest intuicyjny i spójny.

virus update              # wszystkie pakiety

# Środowiska izolowane
virus env myproject       # utwórz (jak python3 -m venv)
virus enter myproject     # aktywuj (jak source activate)
virus exit myproject      # dezaktywuj (jak deactivate)

# Build system
virus run                 # uruchom projekt
virus build               # kompiluj → .bc do /build/
virus build --production  # kompiluj → .hbc do /build/

virus upgrade             # aktualizuj virus
