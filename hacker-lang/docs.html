Interpretowany język programowania dla HackerOS.
Własna składnia operatorowa · Bytecode + Cranelift · Manager pakietów bit · Gen 2

wbudowany w HackerOS tylko dla HackerOS .hl / .bc gen 2 (domyślny)
01

Czym jest Hacker Lang?

Hacker Lang (HL) to interpretowany język programowania napisany w Rust, natywny język skryptowy HackerOS. Pliki źródłowe mają rozszerzenie .hl, skompilowany bytecode — .bc.

✨
Gen 2 (domyślny): typowane zmienne %: typ, arytmetyka $(), pipe do zmiennej |>, for-in @ item in, while ?~, switch ? switch, HackerOS API ||, goroutines z nazwą :* nazwa def, manager pakietów bit z progress barem.

Szybki start

hello.hl
#!/usr/bin/env hl
/// Mój pierwszy skrypt
using <gen 2>

// curl
# <main/colors>

% name: str = HackerOS
% count: int = 42

$( @count * 2 ) -> @doubled

~> Witaj w @name! count=@count doubled=@doubled

> uname -r |> @kernel
~> Kernel: @kernel

@ tool in curl git python3
    ::which @tool
    ? ok
        ::green ✓ @tool
    done
done
URUCHAMIANIE
hl hello.hl              # uruchom
hl compile hello.hl      # → hello.bc (bytecode)
hl compile hello.bc      # → binarka ELF
./hello.bc               # uruchom .bc bezpośrednio
hl repl                  # REPL interaktywny

15

Arytmetyka — $( ) (gen 2)

$( expr ) -> @var
$( 2 + 2 )              -> @res
$( 10 * @count )        -> @mul
$( @a + @b )            -> @sum
$( (2 + 3) * 4 )        -> @paren
$( 100 / @cores )       -> @per_core
% threads: int = $( @cores * 2 )
💡
Backend: sh -c 'echo $(( expr ))'. Obsługuje wszystkie operatory shell: + - * / % ** << >>.

Pipe do zmiennej — |> (gen 2)

> cmd |> @var
> hostname           |> @host
> uname -r           |> @kernel
> date +%Y-%m-%d     |> @today
> id -un             |> @user
> nproc              |> @cores
^> id -u              |> @uid   ;; sudo pipe
~> Host: @host | Kernel: @kernel

For-in — @ item in (gen 2)

@ item in lista
@ tool in curl git python3
    ::which @tool
    ? ok
        ::green ✓ @tool
    done
done

;; Ze zmiennej (ważne: ścieżki z / muszą być w zmiennej)
% dirs = "/tmp /etc /usr"
@ d in @dirs
    ~> @d
done
💡
Iterable to lista oddzielona spacjami. Ścieżki zawierające / należy umieszczać w zmiennej (% dirs = "/tmp /etc"), a nie bezpośrednio w @ item in.

Projekt mode — bit

run.hl + kod

bit → uruchamia interpretowalnie przez hl run run.hl

build.hl + kod

bit → kompiluje do bc/elf/so wg BIT_BUILD_TARGET

source-code/ + build.hl

bit → kompiluje cały katalog (rust-like)

% BIT_BUILD_INPUT  = main.hl

.cache/ tworzony automatycznie i usuwany po zakończeniu — izolowane środowisko.

CLI — hl

WSZYSTKIE KOMENDY
hl plik.hl               # uruchom skrypt
hl run plik.hl           # jawna forma
hl run plik.bc           # uruchom bytecode
hl compile plik.hl       # → plik.bc

hl compile --shared plik.bc  # → .so
hl check plik.hl         # linter + składnia
hl check --meta plik.hl  # + gen + shebang
hl ast plik.hl           # AST jako JSON
hl repl                  # REPL interaktywny
hl shell                 # HL jako powłoka
hl exec nazwa            # skrypt systemowy
hl search fraza          # szukaj skryptów
hl gen-info plik.hl      # gen + shebang
hl docs                  # dokumentacja TUI
hl version               # wersja
hl -c "~> Hej!"          # kod inline

Kompilacja — .hl → .bc → ELF

.hl → .bc

Parsuj, serializuj AST do JSON bytecode. Plik .bc ma shebang i jest wykonywalny.

.bc → ELF

Wczytaj .bc, lower → Cranelift → C runtime → binarka ELF x86_64.

PIPELINE
hl compile skrypt.hl   # → skrypt.bc (bytecode)
./skrypt.bc            # uruchom bezpośrednio
hl compile skrypt.bc   # → skrypt (ELF)
./skrypt               # uruchom binkarkę
