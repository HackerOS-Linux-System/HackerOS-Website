Hacker Lang

Wydajny, interpretowany język programowania dla HackerOS (Debian-based). Własna składnia, własna powłoka, statycznie kompilowana binarka hl.

Rust Interpreted HackerOS Cargo Workspace .hl files Static Binary
01

Czym jest Hacker Lang?

Hacker Lang (HL) to interpretowany język programowania napisany w Rust, zaprojektowany jako natywny język skryptowy systemu HackerOS (oparty na Debianie Testing/Stable). Pliki źródłowe mają rozszerzenie .hl.

HL łączy własną składnię operatorową z bezpośrednim dostępem do powłoki systemowej — wszystko z naciskiem na bezpieczeństwo (izolacja namespace, blokada echo) i wydajność (zoptymalizowany interpreter w Rust).

source.hl ──► Lexer ──► Parser (AST) ──► Executor ──► System
02

Instalacja

Wymagania: Rust 1.75+, Cargo, Linux (Debian/HackerOS).

BASH — build z źródeł
# Klonowanie repozytorium
git clone https://github.com/hackeros/hacker-lang
cd hacker-lang

# Build release (LTO, stripped binary)
cargo build --release

# Instalacja do /usr/local/bin
sudo cp target/release/hl /usr/local/bin/hl
sudo chmod +x /usr/local/bin/hl

# Weryfikacja
hl version
💡
Profil release używa lto = true, codegen-units = 1 oraz strip = true — binarka hl jest statycznie zlinkowana i nie wymaga żadnych zewnętrznych bibliotek HL.
03

Szybki start

hello.hl
/// Mój pierwszy skrypt w Hacker Lang

% name = HackerOS
% version = 0.1.0

:: Witaj w @name v@version!

: greet def
    :: System jest gotowy.
    > uname -r
done

-- greet

? ok
    :: Wszystko działa poprawnie!
done
BASH — uruchomienie
hl hello.hl              # uruchom skrypt
hl shell                 # interaktywna powłoka REPL
hl check hello.hl        # sprawdź składnię
hl ast hello.hl          # wydrukuj AST (JSON)
hl -c ":: Hej!"          # jednolinijkowy kod
hl --help                # pomoc
04

Wyjście — ::

Operator :: to jedyny sposób wypisywania tekstu na stdout w Hacker Lang. Odpowiednik echo z powłoki Bash — ale echo jest tu zakazane wewnątrz bloków komend (>, >> itd.).

🚫
Zakaz echo: użycie > echo tekst spowoduje błąd interpretera. Zawsze używaj :: tekst do wypisywania komunikatów.
SKŁADNIA
:: Zwykły tekst
:: Wartość zmiennej: @nazwa_zmiennej
:: Host: @target, Port: @port

Tekst po :: może zawierać odwołania do zmiennych w postaci @nazwa — zostaną one podstawione automatycznie.

05

Komendy — > ^ ->

Hacker Lang oferuje sześć trybów uruchamiania komend systemowych, różniących się uprawnieniami i poziomem izolacji:

> komenda

Zwykłe wykonanie komendy. Dziedziczą środowisko powłoki.

^> komenda

Wykonanie z sudo. Wymaga uprawnień administratora.

-> komenda

Izolowany namespace (Linux unshare). Oddzielny mount/pid/net.

^-> komenda

Izolowany namespace + sudo. Maksymalna izolacja z uprawnieniami.

>> komenda @var

Komenda z interpolacją zmiennych @nazwa. Zmienne są podstawiane przed wykonaniem.

^>> komenda @var

Interpolacja zmiennych + sudo.

PRZYKŁADY KOMEND
% target = 192.168.1.1
% port = 22

> ls -la /tmp             ;; zwykła komenda
^> apt-get update          ;; sudo
-> curl -s http://test.internal   ;; izolacja sieciowa
^-> nmap -sS 10.0.0.0/24   ;; izolacja + sudo
>> nmap -p @port @target   ;; ze zmiennymi
^>> ssh @user@@target     ;; zmienne + sudo
🔒
Izolacja: tryby -> i ^-> używają unshare --mount --pid --net --fork — komenda uruchamiana jest w oddzielnych przestrzeniach nazw Linuksa, bez dostępu do sieci hosta.

Zmienne — % i @

Operator Rola Przykład
% name = val Deklaracja zmiennej % target = 10.0.0.1
@name Odwołanie / interpolacja :: Host: @target
TYPY WARTOŚCI
% host    = 192.168.1.1      ;; string
% port    = 8080              ;; liczba
% debug   = true              ;; bool
% path    = /tmp/audit/@host  ;; interpolowany string

:: Skanowanie @host:@port → @path

Interpreter automatycznie definiuje kilka wbudowanych zmiennych:

ZmiennaWartość

@HL_OSHackerOS/Debian

@argcLiczba argumentów skryptu

07

Funkcje — : i --

Funkcje definiuje się słowem kluczowym :, po którym następuje nazwa i def. Ciało funkcji kończy się słowem done. Wywołanie to -- nazwa — procesor wykonuje skok do adresu funkcji.

SKŁADNIA FUNKCJI
: perform_audit def
    :: Rozpoczynam skanowanie @target...

    >> nmap -sV -T4 @target

    ? ok
        :: Skan zakończony sukcesem.
    done

    ? err
        :: Błąd podczas skanowania!
    done
done

;; Wywołanie funkcji:
-- perform_audit
⚡
Wydajność: funkcje są przechowywane w hashmap interpretera — wywołanie -- nazwa to bezpośrednie wyszukanie O(1) w tablicy.
08

Logika — ? ok / ? err

Bloki warunkowe sprawdzają kod wyjścia ostatnio wykonanej komendy. ? ok wykonuje blok gdy exit code = 0, ? err gdy exit code ≠ 0.

PRZYKŁAD LOGIKI
> ping -c 1 192.168.1.1

? ok
    :: Host jest online ✓
    >> nmap -sV @target
done

? err
    :: Host niedostępny ✗
    :: Sprawdź połączenie sieciowe.
done
KonstrukcjaWarunek wykonania
? ok ... doneOstatnia komenda zwróciła exit 0
? err ... doneOstatnia komenda zwróciła exit > 0

10

Zależności — //

Operator // (bez domknięcia \\) deklaruje zależność systemową. Interpreter sprawdza, czy narzędzie jest dostępne w PATH — jeśli nie, automatycznie próbuje je zainstalować.

DEKLARACJA ZALEŻNOŚCI
// nmap       ;; wymagany nmap
// curl       ;; wymagany curl
// whois      ;; wymagany whois
// john       ;; John the Ripper

Kolejność instalacji

Interpreter próbuje instalować zależności w tej kolejności:

  1. apt-get -y install <pakiet>
  2. sudo apt-get -y install <pakiet>
  3. sudo lpm install <pakiet> (HackerOS package manager)
📦
Automatyczna instalacja wymaga połączenia z internetem i odpowiednich uprawnień. Jeśli żaden menedżer pakietów nie jest dostępny, wyświetlany jest komunikat błędu — ale wykonanie skryptu jest kontynuowane.

CLI — binarka hl

WSZYSTKIE POLECENIA hl

hl shell                  # interaktywna powłoka REPL
hl check PLIK.hl          # sprawdź składnię (exit 0 = OK)

Przykłady

Audyt sieciowy

network_audit.hl
/// Network Audit Tool — HackerOS

// nmap
// curl
// whois

% target     = 192.168.1.1
% output_dir = /tmp/audit
% ports      = 1-1024

: check_host def
    :: Sprawdzam dostępność @target...
    >> ping -c 1 -W 2 @target

    ? ok
        :: Host @target ONLINE ✓
    done
    ? err
        :: Host @target nie odpowiada ✗
    done
done

: port_scan def
    :: Skanuję porty @ports na @target...
    >> nmap -sV -T4 -p @ports @target

    ? ok
        :: Skan zakończony pomyślnie.
    done
done

: isolated_probe def
    :: Sondowanie w izolacji sieciowej...
    ->> curl -s --max-time 5 http://@target
done

> mkdir -p @output_dir

-- check_host
-- port_scan
-- isolated_probe

:: Audyt zakończony. Wyniki w @output_dir

System update

update.hl
/// Aktualizacja systemu HackerOS

:: Aktualizacja systemu...
^> apt-get update

? ok
    ^> apt-get -y upgrade
    ? ok
        :: System zaktualizowany ✓
    done
done

? err
    :: Błąd aktualizacji. Sprawdź połączenie.
done
